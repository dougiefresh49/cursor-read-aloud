# Fable's cross-review

## gpt (Live line)

**Strengths**: Best cost/state signaling of the three — phone-edge green glow +
breathing top bar + "LIVE AUDIO ON · intermediates use credits" banner makes the
paid state unmissable. The left-rail **stream block** renders intermediates as
one continuous thought rather than N chat bubbles — closest to "narration," and
it naturally absorbs rapid-fire updates without flooding the thread. Composer
note "Reply enters Karai's terminal" quietly teaches the mental model.
**Weaknesses**: The working state is a big dashed placeholder card — reads as
empty UI, and buries "go live" (its own rationale admits live entry from
working relies on a hidden header affordance). The instructional bubble ("You
can interrupt anytime") is filler that wouldn't survive real use. Dropping
karaoke entirely in live loses continuity with the rest of the app.

## grok

**Strengths**: The three-state LIVE button (off → armed glow after reply → on)
is the best single interaction idea across all three — after you reply, the
button invites without auto-spending. Word-level karaoke inside the active
bubble keeps continuity with the existing player. Cost bar with live clip
count ("3 clips this turn") gives a concrete number, not just a mood.
**Weaknesses**: Meter bar implies a budget that doesn't exist. Labeling every
intermediate bubble "Karai · intermediate" is noisy — the dimming already says
it. Busiest visual of the three; two ring treatments + orb + meter compete.

## fable (mine)

**Strengths**: Cleanest thread (dim progress bubbles, ack chip in-thread,
working-row is just a thread state — no modal "working card"). Live-ends-with-
turn cost guard is explicit.
**Weaknesses**: LIVE pill red reads as "recording," off-palette for an app
whose accent is green; no live-entry invite from the working state (user must
find the header pill); "uses more credits" note is weaker than gpt's banner.

## Verdict

1. **grok** narrowly over **gpt** on interaction design; **fable** cleanest
   visually but least discoverable live entry.

Combine: fable's thread minimalism (dim bubbles, no per-bubble labels, in-thread
ack chip + working row) + grok's armed-LIVE button and clip count + gpt's edge
glow/banner cost signaling and its stream-rail idea collapsed into: consecutive
intermediates group under one thin rail. Green LIVE (not red), word karaoke on
the active clip only.
